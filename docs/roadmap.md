# Roadmap

This is a thinking roadmap, not an implementation schedule.

## Track 1: product definition

- sharpen the user promise
- define the delegation model
- define the first mainstream-safe operator behaviors

## Track 2: architecture

- place the operator in the AOSP stack
- define the capability broker
- define the memory and policy layers
- map the extension model

## Track 3: trust and security

- define action policy tiers
- define audit and undo
- define third-party extension boundaries

## Track 4: ecosystem

- define the plugin and SDK story
- define what builders can extend without forking the OS
- define what mainstream users install and understand

## Track 5: market positioning

- map incumbents versus whitespace
- decide whether the wedge is openness, autonomy, trust, extensibility, or all four together
