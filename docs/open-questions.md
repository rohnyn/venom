# Open Questions

## Product

- What is the first user behavior that feels like a real personal operator rather than a better assistant?
- What should the user delegate by default, and what should always stay explicit?
- Should the external story emphasize operator, agent, assistant, or another term entirely?

## UX

- How far can the operator go before the app model starts to feel secondary?
- What is the best default surface: notifications, voice, lock screen, share sheet, or keyboard?
- How should the user review what happened without feeling micromanaged?

## Architecture

- What truly belongs in the OS core versus the extension layer?
- How much app control should rely on native integration versus UI-level agency?
- How should memory be partitioned across local, private, and cloud contexts?

## Security

- What should silent autonomy be allowed to do on day one?
- How should third-party agents be sandboxed?
- What are the minimum audit and rollback guarantees?

## Strategy

- What is the right path from hacker credibility to mainstream adoption?
- What makes this an ecosystem rather than just a product?
