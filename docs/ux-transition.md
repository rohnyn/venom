# UX Transition

## Bridge statement

The right bridge is:

- today: app-first, operator-assisted
- tomorrow: operator-first, app-backed

## Preserve familiar surfaces

Users should still recognize the phone:

- home screen
- notifications
- share sheet
- keyboard
- lock screen
- settings
- recents
- apps

The operator should arrive through those surfaces before it tries to replace them.

## Autonomy ladder

1. Observe
2. Suggest
3. Prepare
4. Execute with confirmation
5. Bounded autonomy
6. Represent

The visible UX can remain relatively stable while the operator's authority increases underneath it.

## What should be first-class now

- user memory
- policy
- audit and undo
- event stream
- delegation model
- capability broker

## What should wait

- broad silent autonomy
- open-ended goal pursuit
- deep high-risk action without approval
- chat as the only interface paradigm

## Product principle

Policy outranks model.

Memory outlives model.

Capabilities are independent of model.
