# Roadmap

This is a thinking roadmap.

## Track 1: Positioning

- define Venom as a clone-first platform
- define the OpenClaw lineage cleanly
- define mobile as the first embodiment

## Track 2: Identity Model

- define clone
- define SOUL
- define rules
- define configurable expression
- define what continuity means

## Track 3: Social Model

- define trust graph
- define shared contexts
- define representation
- define disclosure
- define how Venom relates to people, groups, and services

## Track 4: Continuity And Ownership

- define memory
- define history and activity ledger
- define swappability
- define what user-owned means in practice

## Track 5: Embodiment

- define what stays invariant across surfaces
- define what changes across mobile, desktop, browser, wearable, and workflow embodiments
- define what must remain constant across all surfaces
- define where deep integration matters most
